## What each pass taught me

### Pass 1 → Pass 2 (lesson: scope ≠ engagement)

I shipped **how the work runs** (the broker engagement) when the founder asked for **what we'd build** (the product scope). The lesson: separate engagement-shape from build-shape. Both are needed but they're different artifacts.

### Pass 2 → Pass 3 (lesson: doctrine-outward order)

I shipped a build proposal that listed pages and components. The founder corrected: *"I would not build DefendableOS as a website first. I would build it as a trust operating system with a public surface. The order is: Language → Assignment → Receipt → Tribunal → Deed → Trust → UI."* The UI is layer 7 · not layer 1. The build proposal must ENFORCE this order · not just list features.

### Pass 3 → Pass 4 (lesson: COMP SET wins the board)

I shipped the doctrine-outward build but missed the **comp set**. The founder taught: a Class A 5-cap board approval requires positioning the asset against its market comps · NOT describing the build. *"This is not a website assignment. This is a category-definition assignment."* DefendableOS = UNION of 10 multi-billion-dollar markets (Observability + Compliance + Audit + Ratings + Title Insurance + CI/CD + Cybersecurity + CRE Brokerage + AI Evals + Compute Markets). The asset is priced against THAT stack · not against the cost of a website.
