## What each pass taught me

### Pass 1 → Pass 2 (lesson: scope ≠ engagement)

I shipped **how the work runs** (the broker engagement) when the founder asked for **what we'd build** (the product scope). The lesson: separate engagement-shape from build-shape.

### Pass 2 → Pass 3 (lesson: doctrine-outward order)

I shipped a build proposal that listed pages and components. The founder corrected: *"The order is: Language → Assignment → Receipt → Tribunal → Deed → Trust → UI."* The UI is layer 7 · not layer 1.

### Pass 3 → Pass 4 (lesson: COMP SET wins the board)

I shipped the doctrine-outward build but missed the **comp set**. The founder taught: a Class A 5-cap board approval requires positioning the asset against its market comps. *"This is not a website assignment. This is a category-definition assignment."* DefendableOS = UNION of 10 multi-billion-dollar markets.

### Pass 4 → Pass 5 (lesson: ENGAGEMENT MODEL · this is a DISPOSITION not a VENDOR BUILD)

I shipped board-grade with comp set · but still framed as fee-based vendor engagement ($47K hourly + fixed). The founder corrected: *"to be clear we are looking to disposition of an asset · we are engaging brokers for a disposition assignment · your firm would be the exclusive broker on the deal and we would pay a fee at time of closing · per our rep agreement."*

This is the BROKER MODEL · not the vendor model. Broker absorbs the build cost as listing investment · earns commission at closing only. Same model Mr. DefendableOS used for 30 years to close $8B in CRE.
