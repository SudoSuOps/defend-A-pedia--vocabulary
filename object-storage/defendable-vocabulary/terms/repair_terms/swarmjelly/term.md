# SwarmJelly

## Street Definition

"The Jelly model." That's what the Hacks call it at standup. **SwarmJelly** is the in-house cooked 4-billion-parameter model that powers SwarmFixer. It's not the offense brain · it's not the customer's agent · it's the **defense model** that knows how to repair other models' work. Trained in-house. Open weights. Deployed at `whale:11434`. TEMP locked at 0.05.

## CRE Operator Meaning

In CRE every shop has a **specialist appraiser** for the asset class. The industrial broker doesn't call the residential appraiser. The cold-storage broker doesn't call the multifamily comp guy. You call the specialist · because the specialist has seen the comp set · knows the failure modes · knows what the building should pencil at. SwarmJelly is the specialist appraiser for **failed agent outputs**. It's seen 125,000 of them in training. It knows the failure shape on sight.

## DefendableOS Definition

SwarmJelly is a 4B-parameter language model fine-tuned in-house on a 125,000-pair corpus organized as a **7-failure-mode × 5-RJ-task taxonomy** with a 25% per-mode distribution cap. It ships as `swarmjelly-4b:v1.2` (current production tag) and runs at `whale:11434`. It is the model that backs SwarmFixer · every refinery 5-task output is a SwarmJelly inference call at TEMP=0.05. The model is open-weights · MIT-with-receipt-clause license · operators can self-host on a HoneyBox.

## Jr Broker Use

Don't call SwarmJelly directly. Call it through SwarmFixer. The fixer enforces the 5-task contract, the TEMP setting, the taxonomy tagging, and the validator chain. A direct SwarmJelly call bypasses all of that and the receipt won't anchor. If you have a one-off research need, use the `whale:11434/api/generate` endpoint with TEMP=0.05 and document it as research-only · not production · no deed.

## Sr Broker Use

The Sr Hack monitors SwarmJelly health weekly · reads the inference-throughput chart · watches for latency drift · catches model regressions before they ship a bad refinery run. The Sr Hack also signs off on every model version bump (v1.2 → v1.3) · the upgrade requires a 24-hour canary cook against the 7-mode test suite · the canary must show ≥ the prior version's lift distribution before the production tag flips.

## Tribunal Use

- **Rule layer**: All SwarmFixer outputs MUST be generated by a SwarmJelly version on the approved-tag list (C02 model-config compliance)
- **Rule layer**: Inferences not at TEMP=0.05 are rejected at receipt anchoring (C02)
- **Judge layer**: The Tribunal doesn't grade SwarmJelly directly · it grades the REPAIR task output · SwarmJelly's quality is measured by aggregate Repair Lift over a 7-day window
- **Classification impact**: a SwarmJelly version flagged with > 20% PROPOLIS rate over 7 days gets rolled back to the prior tag · auto-rollback

## Failure Modes

| Mode | Description | Tribunal class on fail |
|---|---|---|
| `model_tag_drift` | Receipt cites a SwarmJelly tag not on the approved list | PROPOLIS · reject receipt |
| `corpus_imbalance` | Training corpus exceeds the 25% per-mode cap | JELLY · model quarantine |
| `endpoint_unreachable` | `whale:11434` doesn't respond · refinery halts | n/a · operational alert |
| `weights_mismatch` | Deployed weights don't match the public release hash | PROPOLIS · security incident |
| `silent_temp_override` | Inference runs at TEMP ≠ 0.05 without explicit doctrine waiver | PROPOLIS |
| `version_regression` | New version ships with worse aggregate lift than prior | n/a · auto-rollback |
